## Overview

Privacy-preserving chat layer for LexCore. Real PII never reaches cloud-LLM payloads. Conversation-scoped entity registry anonymizes at ingestion, the LLM works with realistic surrogates, responses are de-anonymized before user display. Full admin configurability, graceful degradation on provider failure, and complete observability via thread_id correlation logging.

**Privacy invariant (v1.0 scope):** No raw PII reaches cloud chat / auxiliary LLM providers. Cloud auxiliary calls (entity resolution, missed-scan, fuzzy de-anon, title generation, metadata extraction) only ever see surrogate-form data. Embedding provider is a deployer choice with documented tradeoff.

**Source PRD:** `docs/PRD-PII-Redaction-System-v1.1.md`

## Milestone Summary

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Chat-time anonymization (not ingestion-time) | Avoids double-anonymization chains, preserves chunk boundaries, enables conversation-scoped entity resolution | ✓ Good — worked well; no chunk-boundary corruption |
| Conversation-scoped entity registries | Cross-conversation surrogate isolation; bounded registry growth | ✓ Good — SC#5 race verified against live Supabase UNIQUE constraint |
| Buffer-and-de-anonymize over stream-and-de-anonymize | Eliminates surrogate-boundary edge cases; enables fuzzy de-anon on complete response | ✓ Good — SSE status events (`anonymizing`/`deanonymizing`) preserved UX; user-visible latency acceptable |
| Two-pass Presidio detection (0.7 surrogate / 0.3 hard-redact) | Single threshold trades off false-pos vs false-neg unsafely | ✓ Good — surrogate bucket and hard-redact bucket cleanly separated |
| Pre-anonymized cloud LLM path | Privacy invariant: no real PII to cloud; defense-in-depth | ✓ Good — egress filter tested across all 3 auxiliary call sites |
| `EMBEDDING_PROVIDER` configurable (local/cloud) | Deployer choice; preserves RAG-02 by default; no auto-re-embedding on switch | ✓ Good — documented in CLAUDE.md Gotchas; EMBED-02 correctly scoped as deployer-managed migration |
| D-48 canonical-only egress scan | Only longest real value per surrogate used for egress, not all variants — prevents false-positive trips on legal vocabulary | ✓ Good — fixed production blocker in Phase 5 gap-closure |
| `pii_redaction_enabled` moved to `system_settings` DB | Admin can toggle without Railway redeploy | ✓ Good — migration 032 applied; admin UI toggle functional |
