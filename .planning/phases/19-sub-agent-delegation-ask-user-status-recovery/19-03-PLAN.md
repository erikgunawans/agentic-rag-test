<objective>
Build the sub-agent inner loop module — the workhorse that the `task` tool (19-04) will dispatch into. This is a near-clone of `run_deep_mode_loop` minus parent-only tools, with a lower round cap, summary fallback at cap (D-11), and a structured-error wrapper that catches uncaught exceptions before they propagate to the parent (D-12).

Purpose: Without this, the `task` tool has nowhere to dispatch. Failure isolation (TASK-05) only exists if the wrapper converts exceptions to structured `_terminal_result` dicts. Privacy invariant (D-21) only holds if the sub-agent reuses the parent's `ConversationRegistry`, not a fresh one.

Output: One new service module (~300 lines), one config field, one integration test file (~430 lines, 10 tests). Independent from 19-02 because the wrapper boundary in 19-04 (chat.py) handles the agent_runs persistence — sub_agent_loop itself does not call agent_runs_service.

D-09 ownership note: this plan owns BOTH halves of D-09 — the EXCLUDED-set filter (`{"task","write_todos","read_todos"}`) AND the retention assertion that `ask_user` IS in the sub-agent's current_tools list. 19-05 previously held a duplicate retention test (Test 7); per revision-1 it was removed and consolidated here, alongside the filter logic, where it logically belongs.
</objective>
