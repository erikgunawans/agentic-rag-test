Accent is NOT used on: panel body, header row, code preview background, terminal output area.

### Glass Rule (strictly enforced)

`CodeExecutionPanel` is a persistent inline panel — NO `backdrop-blur`, NO `.glass` class, NO `bg-*/opacity + blur`. Use solid `bg-card` or `bg-muted/50` surfaces only.

Source: CLAUDE.md ("Glass (backdrop-blur) ONLY on transient overlays. NEVER on persistent panels."); 2026-refresh spec §3.

## Component Inventory

### NEW: `CodeExecutionPanel.tsx`

Location: `frontend/src/components/chat/CodeExecutionPanel.tsx`

Props interface (executor must match exactly):
```typescript
interface CodeExecutionPanelProps {
  toolCallId: string
  code: string
  status: 'pending' | 'running' | 'success' | 'error' | 'timeout'
  executionMs?: number
  stdoutLines: string[]
  stderrLines: string[]
  files: { filename: string; size_bytes: number; signed_url: string }[]
  errorType?: string
}
```

Internal state:
- `codeExpanded: boolean` — default `false` (D-P11-09, collapsed by default)

#### Panel Anatomy (top to bottom)

**1. Panel Header** (always visible, never collapsed)

Layout: `flex items-center gap-2 px-3 py-2 border-b border-border`

Left cluster (left to right):
- Python badge: `inline-flex items-center gap-1 rounded-full px-2 py-0.5 text-[10px] font-medium bg-primary/15 text-primary` — icon: `<Terminal className="h-3 w-3" />` + label "Python"
- Status indicator (see Status Indicator section below)
- Execution time: `text-xs text-muted-foreground tabular-nums` — shows live counter `"0.0s"` during running, freezes at `"{N}ms"` or `"{N}s"` after completion. Hidden when `status === 'pending'`.

Right cluster:
- Code preview toggle button: `text-xs text-muted-foreground hover:text-foreground flex items-center gap-1 transition-colors` — Lucide `<ChevronRight className="h-3 w-3 transition-transform" />` (rotates 90deg when expanded) + label toggles between i18n key `sandbox.showCode` / `sandbox.hideCode`

**2. Code Preview** (collapsible, default collapsed)

Container: `border-b border-border overflow-hidden` — animate height with `transition-[height] duration-200 ease-out`

When expanded:
```
max-h-80 overflow-y-auto
bg-zinc-900 dark:bg-zinc-950
p-3
```

Content: `<pre className="text-xs font-mono text-zinc-100 whitespace-pre-wrap break-all leading-relaxed">`

Copy-to-clipboard button: top-right corner of code block, `absolute top-2 right-2`, Lucide `<Copy className="h-3 w-3" />`, tooltip "Salin kode / Copy code". Button disappears when code is collapsed.

No syntax highlighting library — plain preformatted text. Syntax highlighting is deferred (not in scope, adds bundle weight). Plain zinc-100 text on zinc-900 background matches terminal aesthetic consistently.

**3. Terminal Output Area** (always visible when any stdout/stderr lines exist; hidden when both arrays are empty AND status is not 'error')

Container:
```
max-h-60 overflow-y-auto
bg-zinc-900 dark:bg-zinc-950
px-3 py-2
font-mono text-xs leading-relaxed
```

Autoscroll behavior: scroll to bottom on each new line push during `status === 'running'`. Stop autoscrolling if user has manually scrolled up (detect via `scrollTop + clientHeight < scrollHeight - 8px` threshold). Resume autoscroll on next render if user scrolls back to bottom.

Line rendering:
- stdout line: `<span className="text-green-400">` + line text
- stderr line: `<span className="text-red-400">` + line text
- Lines render in arrival order (mixed stdout/stderr preserved as received)
- Truncation marker line (appears as last line when 50 KB cap hit): `<span className="text-zinc-500 italic">` + i18n key `sandbox.truncated` (see Copywriting section)

Empty state (no lines yet, status is 'running'): show blinking cursor `<span className="animate-pulse text-green-400">▊</span>` at start of empty line to indicate live stream is active.

**4. File Cards Footer** (visible only when `files.length > 0`)

Section label: `text-xs font-medium text-muted-foreground px-3 pt-2 pb-1` — i18n key `sandbox.filesGenerated`

Cards container: `flex flex-col gap-1 px-3 pb-3`

Each file card:
```
flex items-center gap-3
h-11 px-3
rounded-md border border-border bg-card
hover:bg-muted/50 transition-colors
```

Card contents (left to right):
- File icon: Lucide `<FileDown className="h-4 w-4 text-muted-foreground shrink-0" />`
- Filename: `text-sm text-foreground truncate flex-1` — long filenames truncate at end (not middle; simpler, more readable for legal filenames with descriptive suffixes)
- File size: `text-xs text-muted-foreground tabular-nums shrink-0` — formatted as human-readable (e.g., "1.2 MB", "340 KB", "18 B") via inline formatter (no extra library; `(bytes / 1_000_000).toFixed(1) + " MB"` etc.)
- Download button: `<Button size="sm" variant="default" className="shrink-0 h-7 px-2 text-xs gap-1">` + Lucide `<Download className="h-3 w-3" />` + i18n key `sandbox.download`

Download click behavior: fire `apiFetch('GET /code-executions/{execution_id}')` to refresh signed URL, then `window.open(signed_url, '_blank')`. During the refresh roundtrip (~100ms), show a `<Loader2 className="h-3 w-3 animate-spin" />` replacing the Download icon on that specific card. No global loading state.

Multi-card stacking: cards stack vertically with `gap-1` between them. No horizontal wrapping. Up to 10 files visible; beyond 10, section becomes scrollable (`max-h-[220px] overflow-y-auto`).

#### Status Indicator States

| status | Icon | Color | Label (i18n key) |
|--------|------|-------|-----------------|
| `pending` | `<Loader2 className="h-3.5 w-3.5 animate-spin" />` | `text-muted-foreground` | `sandbox.status.pending` |
| `running` | `<Loader2 className="h-3.5 w-3.5 animate-spin" />` | `text-primary` | `sandbox.status.running` |

| `error` | `<XCircle className="h-3.5 w-3.5" />` | `text-red-500 dark:text-red-400` | `sandbox.status.error` |
| `timeout` | `<Clock className="h-3.5 w-3.5" />` | `text-amber-500 dark:text-amber-400` | `sandbox.status.timeout` |

Status label renders as `text-xs` next to the icon, same color class as icon.

#### Error State (status === 'error')

The terminal output area remains the primary error display — stderr lines already show in red. Additionally, when `status === 'error'` and `errorType` is present:

Error pill below terminal area (above file cards, or at bottom of panel if no files):
```
flex items-center gap-2
mx-3 mb-2 px-3 py-2
rounded-md border border-red-500/30
bg-red-500/10 dark:bg-red-500/15
text-xs text-red-700 dark:text-red-300
```

Content: Lucide `<AlertCircle className="h-3.5 w-3.5 text-red-500 shrink-0" />` + error type label (see Copywriting, `sandbox.error.*` keys).

No retry button — stateless per D-P11-06 decision context (the user interacts with the LLM to re-run, not via a button).

#### Loading Skeleton (status === 'pending', before first stdout line)

Skeleton renders inside the terminal output area slot when no lines have arrived:
```
px-3 py-2 space-y-1.5
```

Three shimmer lines using existing `.shimmer` utility from `index.css`:
- Line 1: `h-3 w-3/4 shimmer rounded`
- Line 2: `h-3 w-1/2 shimmer rounded`
- Line 3: `h-3 w-2/3 shimmer rounded`

Reduced-motion: `.shimmer` animation is already suppressed by the `@media (prefers-reduced-motion)` block in `index.css`. Static muted background shown instead.

#### Multiple Panels in One Turn

Stacking: `ToolCallList` renders `CodeExecutionPanel` instances with `gap-6` (24px) between them. Each panel is keyed by `call.tool_call_id`.

Each panel is fully independent — separate status, separate streams, separate file cards.

## Interaction Contracts

### Streaming Lifecycle

```
tool_start (execute_code)
  → CodeExecutionPanel mounts with status='running', empty arrays
  → Header shows spinner + live timer
  → Terminal shows blinking cursor

code_stdout / code_stderr events
  → Lines appended to sandboxStreams Map entry
  → Terminal area autoscrolls to bottom
  → Skeleton hidden once first line arrives

tool_result event
  → Status switches to 'success' / 'error' / 'timeout'
  → Timer freezes at executionMs value
  → File cards appear if files[] non-empty

delta:{done:true}
  → useChatState refetches messages
  → CodeExecutionPanel switches from sandboxStreams buffer to msg.tool_calls.calls[N].output
  → sandboxStreams Map entry cleared
```

### Code Preview Accordion

- Default: collapsed. Header shows `ChevronRight` pointing right.
- Click "Tampilkan kode" / "Show code": panel expands with `transition-[height] duration-200 ease-out`. Chevron rotates 90deg.
- Click "Sembunyikan kode" / "Hide code": panel collapses. Chevron returns to right.
- Code preview state is local — not persisted, not shared between panels.

### File Download Click

1. User clicks "Unduh" button on a file card.
2. Button replaces icon with `<Loader2 animate-spin />` (card-scoped loading).
3. `apiFetch('GET /code-executions/{execution_id}')` called to get fresh signed URL.
4. On success: `window.open(signed_url, '_blank')`. Button icon restores.
5. On failure: button icon restores, a temporary `text-red-500` state on the card for 2s with i18n `sandbox.downloadError` message, then resets.

## Vertical Stack Layout (Multiple Panels)

When an assistant turn contains N `execute_code` calls:

```
[AgentBadge — if multi-agent]        ← unchanged
[CodeExecutionPanel key=tc1.tool_call_id]  ← 24px gap below
[CodeExecutionPanel key=tc2.tool_call_id]  ← 24px gap below
[ToolCallCard for other tools]             ← existing gap-0.5 preserved
[Assistant text bubble]                    ← unchanged
```

The outer `ToolCallList` container changes from `space-y-0.5` to handle mixed rendering:
- Consecutive `CodeExecutionPanel` instances: `gap-6` (24px) between each
- Between last panel and first `ToolCallCard`: `gap-2` (8px)
- Between consecutive `ToolCallCard` instances: existing `gap-0.5` (2px) preserved

Implementation: render two separate lists (sandbox panels, then generic cards) separated by a `<div className="mb-2" />` spacer, rather than trying to mix gap values in a single flex container.

## Live-Stream Buffer Integration

`useChatState` new state:
```typescript
sandboxStreams: Map<string, { stdout: string[]; stderr: string[] }>
```

`CodeExecutionPanel` prop resolution:
- During streaming: read from `sandboxStreams.get(toolCallId)` — `stdoutLines` and `stderrLines` from the Map entry
- After refetch: read from `msg.tool_calls.calls[N].output.stdout_lines` and `.stderr_lines` — Map entry absent after clear

The panel's status prop:
- During streaming (Map entry present): derived from SSE event type — `tool_start` → `'running'`, `tool_result` → derive from `exit_code`/`error_type`
- After refetch: read from `call.status` field (`'success' | 'error' | 'timeout'`) per D-P11-08

## Accessibility

- All interactive elements (chevron toggle, download button, copy button) have `aria-label` values using the same i18n keys as their visible labels.
- Status indicators include `aria-live="polite"` on the status label span — status changes announce to screen readers without interrupting.
- Terminal output area: `role="log"` with `aria-label` of i18n key `sandbox.status.running` during streaming, updating to the final status label after completion.
- File download cards: `role="button"` already implied by `<Button>` usage; `aria-label` = `"Unduh {filename}"` / `"Download {filename}"`.
- Code preview toggle: `aria-expanded={codeExpanded}` on the button.
- Reduced motion: `.shimmer` skeleton animation already suppressed by existing `@media (prefers-reduced-motion)` in `index.css`. Terminal autoscroll: no animation suppression needed (smooth scroll via `scrollIntoView` is a layout operation, not decorative animation).
