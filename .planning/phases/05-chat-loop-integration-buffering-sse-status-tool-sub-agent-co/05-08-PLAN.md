# Plan 05-08: pii_redaction_enabled DB-Backed Admin Toggle

## Objective

Move the PII redaction master toggle from `config.py` (pydantic-settings env var, hardcoded default `True`, never queryable) into the `system_settings` table where every other admin-settable feature flag lives. This:

1. Lets admins toggle PII redaction from the admin UI without a Railway redeploy.
2. Restores the SC#5 invariant testability — production currently always runs ON because there is no off path. The DB column is the off path.
3. Brings PII redaction in line with `tools_enabled`, `agents_enabled`, `rag_hybrid_enabled`, `pii_missed_scan_enabled`, etc. — all toggled via system_settings, all cached with the same 60s TTL.
4. Eliminates the documentation gap in CLAUDE.md (which already describes `pii_redaction_enabled` as a "system setting" — this plan makes the docs accurate).

UAT confirmed (2026-04-28, Test 2): the field is absent from the 44-key admin API response. CLAUDE.md says "Toggle via `pii_redaction_enabled` system setting" — that statement is currently false; this plan makes it true.

### Task 1: Create migration 032

**File:** `supabase/migrations/032_pii_redaction_enabled_setting.sql` (NEW — note: migrations live at the repo root `supabase/migrations/`, NOT `backend/migrations/`. Existing 030 and 031 confirm the path.)

Use `/create-migration` skill or write directly. Content:

```sql
-- Migration 032: PII Redaction System-Settings Column
--
-- Plan 05-08 gap-closure. Moves pii_redaction_enabled from config.py env var
-- (always True default, never admin-queryable) into system_settings as a
-- runtime-toggleable feature flag.
--
-- Default TRUE preserves production current behavior (PII redaction is ON in
-- production after the v0.3.0.0 ship; no operational change).
--
-- system_settings is a single-row table (id=1). RLS already gates writes via
-- the super_admin role check. No new RLS policy needed.

ALTER TABLE system_settings
  ADD COLUMN IF NOT EXISTS pii_redaction_enabled BOOLEAN NOT NULL DEFAULT TRUE;

-- Idempotent backfill — the singleton row already exists.
UPDATE system_settings SET pii_redaction_enabled = TRUE WHERE id = 1;

COMMENT ON COLUMN system_settings.pii_redaction_enabled IS
  'Master toggle for the v1.0 PII Redaction milestone. When TRUE, every chat turn '
  'runs the full Phase 5 redaction pipeline (anonymize → LLM with surrogates → de-anonymize). '
  'When FALSE, behavior is byte-identical to pre-v0.3 baseline (SC#5 invariant). '
  'Replaces the deprecated config.py PII_REDACTION_ENABLED env var (Plan 05-08).';
```
