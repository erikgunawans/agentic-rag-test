# Phase 22 Plan 13: Migration Source Harness SUMMARY

One-liner: Idempotent ALTER TABLE migration that widens `workspace_files_source_check` from `('agent','sandbox','upload')` to `('agent','sandbox','upload','harness')` — closes UAT Gap 1 BLOCKER.

## Objective

The DB CHECK constraint `workspace_files_source_check` (defined in migration 039) rejected `source='harness'` with SQLSTATE 23514. Plan 22-11 had already widened the frontend TypeScript type but never updated the schema. This migration adds the missing DB-side fix so the live harness CR-01 phase can write `contract-text.md` (and any harness-scoped workspace file) without raising a constraint violation.

## Constraint Before and After

**Before (039_workspace_files.sql line 18):**
```sql

```
(constraint name: `workspace_files_source_check` — auto-generated by Postgres)

**After (043_workspace_files_source_harness.sql):**
```sql

```

After applying migration 043 to remote, `pg_get_constraintdef` should return:
```

```
