    <title>Author CodeExecutionPanel.test.tsx — 4 SC#2 cases + 2 edge cases</title>
    <type>execute</type>
    <action>
      Create `frontend/src/components/chat/CodeExecutionPanel.test.tsx` with
      EXACTLY these test cases (D-P16-09):

      Top-of-file imports:
      ```ts
      import { describe, it, expect, vi, beforeEach } from 'vitest'
      import { render, screen, within } from '@testing-library/react'
      import userEvent from '@testing-library/user-event'
      import { CodeExecutionPanel } from './CodeExecutionPanel'

      // D-P16-08: mock the network layer; do NOT spin up MSW.
      vi.mock('@/lib/api', () => ({
        apiFetch: vi.fn(),
      }))
      import { apiFetch } from '@/lib/api'
      ```

      Test 1 — `streaming output renders stdout/stderr lines in order with color`:
      Render the panel with a `sandboxStreams` Map fixture containing
      `stdout: ["line A", "line B"]` and `stderr: ["err X"]`. Assert all three
      lines are in the DOM in that order; assert stdout lines have a green-ish
      class (e.g. `text-emerald-` or whatever the component uses) and stderr
      lines have a red-ish class. Use `screen.getByText` and inspect
      `.className` on the resulting elements.

      Test 2 — `terminal block renders dark background and ANSI-stripped text`:
      Pass a sandboxStream with an ANSI-coded line (e.g. `"[31mhello[0m"`).
      Assert the rendered text is `"hello"` (no escape codes visible) and the
      enclosing terminal block has a dark-bg class (`bg-zinc-900` or similar).

      Test 3 — `Download button calls apiFetch and exposes signed URL`:
      Render the panel with `executionId="exec-abc"` and a status that allows
      download (e.g. `completed` with non-empty output_files). Mock
      `apiFetch` to resolve to `{ signed_url: "https://signed.example/foo" }`.
      Click the Download button. Assert `apiFetch` was called with a path
      containing `/code-executions/exec-abc` and that an `<a>` element exists
      whose `href` ends in `/foo` OR that `window.open`/`location.href` was
      invoked with that URL (D-P16-10: do NOT assert actual navigation in
      jsdom).

      Test 4 — `history-reconstruction parity with streaming render`:
      Render once with live `sandboxStreams` data, snapshot the rendered DOM
      (`container.innerHTML`). Render a second instance with the same data
      shape passed via `msg.tool_calls.calls[N].output` (persisted shape).
      Assert the two outputs are textually equivalent for stdout/stderr lines
      and status badge content. Acceptable comparison: use
      `expect(textContentNormalized(a)).toBe(textContentNormalized(b))` with
      a small whitespace-collapse helper. Per D-P16-09 #4: same DOM shape,
      same icons, same status badge.

      Test 5 (edge) — `executionId undefined disables Download`:
      Render with `executionId={undefined}` and any other valid props.
      Assert the Download button is in the document AND has `disabled` attr
      OR is not rendered at all (component's choice — assert ONE of the two).
      No `apiFetch` mock should be called.

      Test 6 (edge) — `status timeout renders Clock icon`:
      Render with `status="timeout"`. Assert an SVG element with the
      lucide-react `Clock` icon (a `<svg>` whose `data-lucide` or `class`
      mentions "lucide-clock" / `aria-label`/`title` is "Clock") is present.

      Group all six in a single `describe('CodeExecutionPanel', () => { ... })`.
      Add a `beforeEach(() => { vi.clearAllMocks() })` to reset the apiFetch
      mock between tests.
    </action>
