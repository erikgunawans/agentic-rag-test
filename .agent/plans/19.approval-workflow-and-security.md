# Features 5 & 6: Approval Workflow Engine + MFA & Security Hardening

**Complexity**: ⚠️ Medium — Two features, 3 new DB tables + alter, 3 new routers, 3 new pages

## Feature 5: Approval Workflow Engine

### Migration `015_approval_workflows.sql`
- `approval_workflow_templates`: id, name, description, steps (jsonb array of {step_order, approver_role, approver_email, action: approve/reject/return}), created_by, is_active
- `approval_requests`: id, user_id, template_id, resource_type (document_tool_result), resource_id, title, status (pending/in_progress/approved/rejected/cancelled), current_step, submitted_at, completed_at
- `approval_actions`: id, request_id, step_order, actor_id, actor_email, action (approve/reject/return), comments, acted_at
- RLS: users see own requests, approvers see requests where they're the current step approver, admins see all

### Backend
- `approval_service.py`: submit_for_approval(), get_pending_for_user(), take_action(), advance_workflow()
- `approvals.py` router: POST /approvals/submit, GET /approvals/inbox, GET /approvals/{id}, POST /approvals/{id}/action, GET /approvals/templates (admin), POST /approvals/templates (admin)

### Frontend
- `ApprovalInboxPage.tsx` at `/approvals` — mobile-first card list, filter by status
- "Submit for Approval" button on DocumentCreationPage + ComplianceCheckPage results
- IconRail nav item

## Feature 6: MFA & Security Hardening

### Migration `016_security_hardening.sql`
- ALTER system_settings: add mfa_required boolean, session_timeout_minutes int
- `user_profiles`: id, user_id, display_name, department, is_active, deactivated_at, deactivated_by, last_login_at

### Backend
- `user_management.py` router: GET /admin/users, PATCH /admin/users/{id}/deactivate, PATCH /admin/users/{id}/reactivate

### Frontend
- MFA enrollment section in SettingsPage (Supabase TOTP)
- MFA challenge on login in AuthPage
- Admin user management at `/admin/users`
