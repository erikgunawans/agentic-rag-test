# Feature 3: Enhanced Drafting Workbench

**Complexity**: ⚠️ **Medium** — Multiple tables, two new routers, frontend pages, LLM prompt changes. Single-pass feasible with clear spec.

## Overview

Expand document creation from 4 basic doc types to a full drafting workbench: clause library (reusable clauses with risk scores), document templates, 7 doc types, and per-clause risk scoring in generated output.

## Current State

- 4 doc types: generic, NDA, sales, service (hardcoded in frontend `DocType` union)
- No clause library or template management
- `document_tool_service.py` `create_document()` accepts any `doc_type` string, builds LLM prompt
- Results stored in `document_tool_results` table (JSONB `result` column)
- Confidence scoring + HITL review gates already in place (F2)

## Tasks

### Task 1: Database Migration (`supabase/migrations/014_drafting_workbench.sql`)

**Tables:**

```sql
-- clause_library
CREATE TABLE clause_library (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users(id) NOT NULL,
  title text NOT NULL,
  content text NOT NULL,
  category text NOT NULL,  -- e.g. 'confidentiality', 'termination', 'payment', 'liability', 'indemnity', 'force_majeure', 'dispute_resolution', 'general'
  applicable_doc_types text[] DEFAULT '{}',  -- which doc types this clause applies to
  risk_level text NOT NULL DEFAULT 'medium' CHECK (risk_level IN ('low', 'medium', 'high', 'critical')),
  language text NOT NULL DEFAULT 'id' CHECK (language IN ('id', 'en', 'both')),
  tags text[] DEFAULT '{}',
  is_global boolean NOT NULL DEFAULT false,  -- admin-created, visible to all
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- document_templates
CREATE TABLE document_templates (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users(id) NOT NULL,
  name text NOT NULL,
  doc_type text NOT NULL,
  description text,
  content text,  -- template body/structure hint for LLM
  fields_schema jsonb DEFAULT '[]',  -- array of {name, label, type, required, options?}
  default_clauses uuid[] DEFAULT '{}',  -- clause_library IDs to pre-select
  is_global boolean NOT NULL DEFAULT false,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
```

**RLS:**
- clause_library: users see own + global (`user_id = auth.uid() OR is_global = true`), only own insert/update/delete
- document_templates: same pattern
- Admin (super_admin) can create global clauses/templates

**Indexes:**
- `clause_library(user_id)`, `clause_library(category)`, GIN on `tags`
- `document_templates(user_id)`, `document_templates(doc_type)`

**Triggers:**
- `updated_at` trigger on both tables

**Validation:**
- Run migration via Supabase MCP `apply_migration`
- Verify tables exist and RLS is enabled
- Test insert as authenticated user + read global rows

### Task 2: Backend — Clause Library Router (`backend/app/routers/clause_library.py`)

**Endpoints:**
- `GET /clauses` — list user's + global clauses, optional `?category=X&doc_type=X&search=X`
- `GET /clauses/{id}` — get single clause
- `POST /clauses` — create clause (body: title, content, category, applicable_doc_types, risk_level, language, tags)
- `PATCH /clauses/{id}` — update clause (own only)
- `DELETE /clauses/{id}` — soft or hard delete (own only)
- `GET /clauses/categories` — distinct categories list

**Notes:**
- Search endpoint uses `ilike` on title + content + tags
- Admin can set `is_global: true`
- Pydantic models: `ClauseCreate`, `ClauseUpdate`, `ClauseResponse`
- Register router in `main.py` with prefix `/clauses`
- Audit log on create/update/delete

**Validation:**
- curl: create a clause, list it, search by category, update, delete
- Verify non-owner cannot update/delete another user's clause

### Task 4: Backend — Expand Doc Types + Clause/Template Integration

**Changes to `document_tool_service.py`:**
- Add `clause_texts: list[dict]` param to `create_document()` — each dict has `title`, `content`, `risk_level`
- Add `template_content: str | None` param
- Update LLM system prompt to:
  1. Include selected clauses with their risk levels
  2. Instruct LLM to incorporate clauses into the document
  3. Request `clause_risk_scores` array in JSON response (each clause gets a contextual risk assessment)
- Update `GeneratedDocument` Pydantic model: add `clause_risk_scores: list[dict]` field (each: `clause_title`, `risk_level`, `risk_explanation`)

**Changes to `document_tools.py` router:**
- `POST /document-tools/create` accepts new optional FormData fields:
  - `clause_ids` (JSON string array of clause UUIDs)
  - `template_id` (UUID string, optional)
- Router fetches clause content + template content before calling service
- Stores clause_ids and template_id in `input_params` JSONB

**New doc types** (no backend validation change needed — service accepts any string):
- `vendor` — Vendor/Supplier Agreement
- `jv` — Joint Venture Agreement
- `property_lease` — Property Lease Agreement
- `employment` — Employment Contract
- `sop_resolution` — SOP/Board Resolution

**Validation:**
- curl: create document with clause_ids + template_id, verify clauses appear in output, verify clause_risk_scores in response
- Test each of the 7 doc types
