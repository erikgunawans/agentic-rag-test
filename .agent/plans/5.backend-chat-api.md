# Plan 5: Backend Chat API
⚠️ **Medium** - Thread CRUD + SSE streaming endpoint

## Goal
REST endpoints for thread management and SSE streaming chat.

## Tasks

### 5.1 backend/app/routers/threads.py
Endpoints (all require auth via `get_current_user` dependency):

- `GET /threads` → list all threads for current user (ordered by updated_at desc)
- `POST /threads` → create thread (creates OpenAI thread + DB record), returns thread
- `DELETE /threads/{thread_id}` → delete thread + all messages (cascade)
- `PATCH /threads/{thread_id}` → update thread title

Thread DB model:
```python
{
  "id": "uuid",
  "title": "string",
  "openai_thread_id": "string",
  "last_response_id": "string | null",
  "created_at": "timestamp",
  "updated_at": "timestamp"
}
```

### 5.2 backend/app/routers/chat.py
- `POST /chat/stream` → SSE streaming endpoint
  - Body: `{"thread_id": "uuid", "message": "string"}`
  - Auth required
  - Fetches thread from DB, validates ownership
  - Saves user message to `messages` table
  - Calls `openai_service.stream_response()`
  - Accumulates full response text
  - Saves assistant message to `messages` table
  - Updates `threads.last_response_id`
  - Returns `StreamingResponse(media_type="text/event-stream")`

### 5.3 Pydantic request/response models
```python
class CreateThreadRequest(BaseModel):
    title: str = "New Thread"

class SendMessageRequest(BaseModel):
    thread_id: str
    message: str

class ThreadResponse(BaseModel):
    id: str
    title: str
    created_at: datetime
    updated_at: datetime
```

## Validation
- `POST /threads` with valid JWT → returns thread object
- `GET /threads` → returns list
- `POST /chat/stream` → streams SSE events ending with `done: true`
- Check DB: messages saved after chat call
