# Plan 6: Frontend Auth
⚠️ **Medium** - Supabase Auth with protected routes

## Goal
Supabase auth flow (login/signup), JWT management, typed API client.

## Tasks

### 6.1 frontend/src/lib/supabase.ts
- Create Supabase client with `createClient(VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY)`
- Export typed client

### 6.2 frontend/src/lib/api.ts
- `apiFetch(path, options)` wrapper that:
  - Gets JWT from `supabase.auth.getSession()`
  - Adds `Authorization: Bearer <token>` header
  - Prefixes with `VITE_API_BASE_URL`
  - Throws on non-2xx responses

### 6.3 frontend/src/lib/database.types.ts
- TypeScript types for threads and messages tables
- `Thread`, `Message` interfaces matching DB schema

### 6.4 frontend/src/components/auth/AuthGuard.tsx
- Checks `supabase.auth.getSession()`
- Subscribes to `supabase.auth.onAuthStateChange()`
- Renders children if authenticated, redirects to `/auth` if not
- Shows loading spinner during initial check

### 6.5 frontend/src/pages/AuthPage.tsx
- Tab switcher: Login / Sign Up
- Email + password form
- Calls `supabase.auth.signInWithPassword()` or `supabase.auth.signUp()`
- Shows error messages
- Redirects to `/` on success

### 6.6 frontend/src/App.tsx
- React Router routes:
  - `/auth` → `<AuthPage />`
  - `/` → `<AuthGuard><ChatPage /></AuthGuard>`
- Auth state listener → redirect to `/auth` on signout

## Validation
- Navigate to `/` → redirects to `/auth` when not logged in
- Sign up with email/password → redirected to chat page
- Refresh page → stays on chat page (session persisted)
- Sign out → redirected to `/auth`
