# Plan 3: Backend Core
⚠️ **Medium** - FastAPI app foundation

## Goal
Create FastAPI app with Pydantic settings, Supabase client factory, and JWT auth dependency.

## Tasks

### 3.1 backend/app/config.py
- `Settings` class extending `pydantic_settings.BaseSettings`
- Fields: SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, OPENAI_API_KEY, OPENAI_VECTOR_STORE_ID (optional), LANGSMITH_API_KEY (optional), LANGSMITH_PROJECT, LANGCHAIN_TRACING_V2
- Load from `.env` file

### 3.2 backend/app/database.py
- `get_supabase_client()` → supabase-py `Client` using service role key
- `get_supabase_authed_client(token: str)` → `Client` with user JWT for RLS

### 3.3 backend/app/dependencies.py
- `get_current_user(token: HTTPAuthorizationCredentials)` → validates JWT via Supabase
- Returns user dict `{id, email}` or raises 401
- Uses `get_supabase_client().auth.get_user(token)`

### 3.4 backend/app/main.py
- FastAPI app with CORS (allow `http://localhost:5173`)
- Include routers (threads, chat)
- `GET /health` → `{"status": "ok"}`
- Startup: configure LangSmith if key present

### 3.5 backend/requirements.txt

### 3.6 backend/run.sh
```bash
#!/bin/bash
source venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## Validation
- `cd backend && source venv/bin/activate && pip install -r requirements.txt`
- `bash run.sh` → server starts
- `curl http://localhost:8000/health` → `{"status":"ok"}`
