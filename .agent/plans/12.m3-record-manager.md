# Module 3: Record Manager

✅ **Simple** — Single-pass executable, low risk

## Goal

Prevent duplicate ingestion by computing a SHA-256 hash of uploaded file bytes and skipping re-processing when the same content already exists for the user.

## Tasks

### 1. Database Migration
- **File**: `supabase/migrations/004_record_manager.sql`
- Add `content_hash text` column to `documents` (nullable)
- Add partial index `(user_id, content_hash) WHERE content_hash IS NOT NULL`
- **Validation**: Apply in Supabase dashboard; verify column + index exist

### 2. Backend — Upload endpoint dedup
- **File**: `backend/app/routers/documents.py`
- Add `import hashlib`
- After reading `file_bytes`: `content_hash = hashlib.sha256(file_bytes).hexdigest()`
- Query `documents WHERE user_id = :user AND content_hash = :hash`
- completed match → return 200 `{..., duplicate: True}`, skip all side effects
- pending/processing match → raise 409
- failed match → delete failed record (storage + DB), proceed fresh
- Include `content_hash` in `documents.insert()`
- New uploads return `duplicate: False` in response
- Update `list_documents` select to include `content_hash`
- **Validation**: Upload same file twice → 200 + duplicate:true on second upload

### 3. Frontend — Duplicate feedback
- **File**: `frontend/src/components/documents/FileUpload.tsx`
- Add `info` state; read response JSON; show blue info text on `duplicate: true`
- **File**: `frontend/src/lib/database.types.ts`
- Add `content_hash: string | null` to `Document` interface
- **Validation**: Upload same file twice → blue info message appears

### 4. API Tests
- **File**: `tests/api/test_documents.py`
- Add `import uuid`
- Add `TestDocumentDedup` class with 5 tests (unique UUID content per test):
  - DEDUP-01: new upload → duplicate:false, 202
  - DEDUP-02: re-upload completed → duplicate:true, 200, same id
  - DEDUP-03: same name different content → 202 each
  - DEDUP-04: content_hash in GET /documents (64-char hex)
  - DEDUP-05: delete then re-upload → 202, new id
- **Validation**: `pytest tests/api/test_documents.py -v`
