## Task 4: Deploy backend to Railway

1. Install Railway CLI: `npm i -g @railway/cli`
2. `railway login`
3. `railway init` (or link to existing project)
4. Set all env vars from `.env` via `railway variables set`
5. `railway up` from the `backend/` directory
6. Get the public URL from Railway dashboard

**Validation**: `curl https://<railway-url>/health` returns `{"status": "ok"}`

---

## Task 5: Deploy frontend to Vercel

1. Use Vercel MCP `deploy_to_vercel` tool (or `npx vercel` from `frontend/`)
2. Set environment variables:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   - `VITE_API_BASE_URL` = Railway backend URL
3. Deploy to production

**Validation**: Visit Vercel URL â†’ login page loads â†’ can sign in and chat.
